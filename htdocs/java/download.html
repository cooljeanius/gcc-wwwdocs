Downloading GCJ

Current Release

The current source release is still provided in two parts: the compiler, gcj, and the runtime, libgcj. Some time in the future, a single source release of gcc will contain everything you need.

Gcj is part of the Gnu Compiler Collection, GCC. Follow the links on http://gcc.gnu.org/releases.html to download the compiler. You will need either the full distribution, or each of the Core Compiler, C++ and Java distributions.

The most current official release of libgcj is 2.95.1. It's broken. Don't use it. For the records we'll just say it's available from ftp://sources.redhat.com/pub/java/libgcj-2.95.1.tar.gz, but you haven't heard that from us.

Development Sources

The latest development sources are available in three forms: periodic snapshots, prebuild RPMs and directly from the cvs source repository.

Note that the current development sources are generally for "bleeding edge" developers, and those who need features not found in older, more stable, versions. Also note that the current development sources may have bugs which prevent it from compiling or running; however, these things should be fixed immediately. Please submit a problem report if you are using the development sources and run into such a problem. If you are going to build from source you want to read the Build Issues FAQ.

Snapshots

Snapshots are made about once a week. They are available from http://gcc.gnu.org/snapshots.html. Starting from 12/11/2000, they contain everything you need (compiler + java runtime.)

RPM

  • Red Hat Linux 7 includes relatively recent, "QA'ed" snapshots of the gcj and libgcj development trees which should work out-of-the-box. Look for the gcc-java, libgcj, and libgcj-devel RPM packages on disc 2.

RPMs are also made available occasionally by Anthony Green. The are available from ftp://sources.redhat.com/pub/java/rpm/ and Mirror Sites.

Everything will install under /usr/local/ag. The source RPM unpacks as a checked out anonymous source tree with GCC and libgcj unified under a single root. Debian users can use alien --to-deb to convert the rpms.

You have to add /usr/local/ag/bin to your PATH and add /usr/local/ag/lib to your LD_LIBRARY_PATH.

cvs

The easiest way to get libgcj is to check it out from the gcc cvs repository. Here's how perform an anonymous read-only CVS access: 
cvs -d:pserver:anoncvs@gcc.gnu.org:/cvs/gcc login
  (password is ``anoncvs'')
cvs -z9 -d:pserver:anoncvs@gcc.gnu.org:/cvs/gcc co gcc

Here's how to perform a read-write CVS access, provided that you're were givin a account on gcc.gnu.org: 

cvs -z 9 -d :ext:username@gcc.gnu.org:/cvs/gcc co gcc
