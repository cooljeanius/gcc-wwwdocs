Testing GCJ

General Runtime Tests

Runtime tests are executed via 'make check' from the testsuite directory of the libjava hierarchy in the build tree. Additional runtime tests can be checked into the tree. Regression tests of the core packages in libgcj are run automatically via the Mauve test suite (described below) if it is made available when the tests are being run.

Mauve

The Mauve Project is a project to write a suite of tests for the Java Class Libraries. We encourage developers to contribute test cases to Mauve.

libgcj supports Mauve testing. If you have Mauve checked out in, e.g., ~/mauve/, then you can tell the libgcj to run the Mauve tests like so: make MAUVEDIR=~/mauve check.

To highlight regressions, a mechanism has been set up in mauve.exp to track expected failures by comparing against the contents of libjava/testsuite/libjava.mauve/xfails from the source hierarchy. This file should be updated as new (failing) tests are added to Mauve or when fixes are made to libgcj that correct for failing Mauve tests.

Jacks

The Jacks project exists to create a test suite for Java compilers. We encourage developers to contribute test cases to Jacks.

Changes affecting the gcj front end can usefully be tested against Jacks. We don't as yet have an automated mechanism to run Jacks tests in our test framework.
