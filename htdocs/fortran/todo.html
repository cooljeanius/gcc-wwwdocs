G95: Open Projects

This list is probably not complete, but it seems long enough already :-) Some of the items are also filed as a bug report in our bugzilla problem report database as indicated below.

Compiler driver

  • Call f951 for files ending in .f (using -ffixed-form).
  • Call the C preprocessor for files ending in .F, .F90 and .F95.
  • Remove or redefine -fg77-calls.

Parser

  • Give NULL() pointer correct attributes.
  • Fix array constructor handling in expressions and intrinsics.
  • Complete implementation of section 14.1.2: Procedure type resolution rules.
  • Implement resolution of COMMON blocks and EQUIVALENCE associations.
  • Fix other issues in the type/name resolution phase.
  • Improve diagnostics, some of them are cryptic or point to the wrong place.
  • Improve error printing for diagnostics within simplification functions.
  • See about making Emacs-parsable error messages.
  • Don't bother storing non-parameter variable initializers in modules.
  • Check for improper (full section) use of assumed size arrays.
  • Do something about the expr->symbol vs expr->value.function.esym mess.

Middle-end Interface

  • Support for arrays of character strings (PR12842).
  • Support for character string array constructors (ditto).
  • Implement functions returning derived types.
  • Support alternate ENTRY points (PR13082).
  • Correctly initialize POINTER variables.
  • Handle constant initializers for derived types.
  • Handle character string parameters.
  • Implement DATA.
  • Implement character variables in COMMON and EQUIVALENCE blocks.
  • Finish compiler side of intrinsic functions.
  • Produce debugging information.
  • Improve generated code (lots of room for that).
  • Don't create array descriptors where we don't need them (temporary arrays).
  • Support large (unexpanded) array constructors as variable initializers.
  • Use of array parameters in array declarations.
  • Teach the scalarizer bounds calculation code about functions.

Run time library

  • Complete the implementation of direct access I/O (PR12921).
  • Complete the implementation of derived type I/O (PR13278 and PR13280).

g77 compatibility

  • Implement Hollerith data.
  • Implement the -fugly- family of compiler options.
  • Fix g77/f2c compatibility for returning complex numbers, and other wierdness.

Testsuite

  • Figure out what causes the remaining testsuite parse failures.
  • Add more testcases to the execution test suite.
  • Add negative tests (ie check invalid code is rejected).

Documentation

  • Finish (start?) infopages and other documentation.
  • Split out an "invoke.texi" so that a man page can be generated automatically.
  • Maintain web page.
