Rolling a GCC Release

  1. Before rolling the release, update the release notes web pages and ensure that they are all listed in contrib/gennews.
  2. For a new major release, ensure that the build status page is present.
  3. Update the releases.html web page.
  4. Update the version numbers of the current and future releases on the main web page, and add a proper news item there as well.
  5. Generate online documentation for the new release with update_web_docs and link to it from onlinedocs/index.html (but don't break URLs to documentation for previous releases even if you remove the links to it).
  6. Add the release tag to the list in cvs.html (the branch tag should already be there).
  7. Update ~ftp/pub/gcc/README and ~ftp/pub/gcc/releases/index.html.
  8. Upload the release to ftp.gnu.org
  9. Announce the release the gcc-announce@gcc.gnu.org and gcc@gcc.gnu.org mailing lists.
    • Make sure that most mirrors have retrieved the new release already.
    • In the case of a major release, coordinate announcements with the FSF.
  10. Increment the version numbers in the [vV]version.c files and make them include the date and (prerelease) instead of saying (release) and the old release version, before allowing any new checkins to the branch.
