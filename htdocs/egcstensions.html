egcs extensions

This is a list of (experimental) extensions to egcs which, for one reason or the other, did not (yet) make it into the official source tree. Please direct feedback and bugreports to their respective maintainers, not our mailing lists.

PentiumGCC

The PentiumGCC (short PGCC) is an extension of egcs specifically aimed at the newer Intel chips and their clones (Pentium, PPro, Pentium-II and the Cyrix6x86 and AMD-K6+ chips). The current goal is to enhance egcs to a point where it generates code as fast as PGCC, but until that work is finished PGCC is still maintained, and often achieves substantial speed improvements over egcs and gcc.

ATMEL AVR port

This is an experimental port for the 8bit ATMEL AVR micro controllers.

Patch to build elsewhere than in the source directory using Sun make

Contact: hp@axis.se

By means of a patch for GCC 2.95.1 Sun make should be able to bootstrap the compiler even when using Sun make, which is not sufficiently standards compliant, for a srcdir != objdir setup.

Apply it with (for example) GNU patch:

patch -p1 -d /path/to/gcc-2.95.1 < gcc-2.95.1-sun-make-patch1.txt
This patch is not supposed to interfere with proper building and installation on non-Sun systems or when using GNU make.
